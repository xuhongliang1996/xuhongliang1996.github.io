Java 多线程

线程基础

实现方式

在Java中, 实现多线程的方式有两种, 继承Thread类或实现Runnable接口, 然后重写run方法.

这两种方式创建线程后在工作时的性质是一样的, 因为Java不支持多继承, 所以推荐使用后者.

后者需要通过Thread对象的构造方法来辅助构造线程, 线程的运行顺序是异步的.

优先级

Java中, 每个线程都会拥有一个默认的优先级(5), 如果未定义, 则会继承父类的优先级.

(优先级高的并不一定会先执行, cpu会尽量将资源分配给优先级高的, 还是具有一定的随机性.)

守护线程

Java中, 线程一般分为用户线程(user)和守护线程(Daemon).

守护线程一般用于为其他类别的线程提供服务, 当用户线程都退出时, 守护线程也会退出.

最简单的守护线程例子是JVM中的垃圾回收线程

线程状态

线程的状态有五种, 状态信息保存在Thread.State枚举类.

  1. 新建(NEW): 线程被创建后的初始状态

  2. 就绪(RUNNABLE):调用start方法, 并获取到cpu分配时间时进入运行状态.

  3. 等待/阻塞/挂起(WAITING/BLOCKED/TIMED_WAITING)

    a. 线程需要等待某个条件时, 而条件未成熟时, 等待调度器通知时, 进入等待状态.

    b. 获取某个资源时, 资源被加锁, 等待资源被释放时, 进入阻塞状态.

    c. 线程需要等待一些时间时, 进入挂起状态.

  4. 死亡(TERMINATED): run方法执行完毕或因异常导致退出, 线程死亡.

NEW, RUNNABLE, TIMED_WAITING, TERMINATED状态.

常用方法

启动/休眠相关

  • start: 启动线程
  • sleep(n): 让当前正在执行的线程休眠n毫秒
  • isAlive: 判断线程是否在活动状态(线程已经启动且尚未停止)
  • getId: 取得线程的唯一标识

停止相关

  • interrupted: 测试当前线程是否有中断标记并清除中断标记
  • isInterrupted: 测试线程是否已经中断标记(并不清除中断标记)
  • interrupt: 设置中断标记.

切换相关

  • yield: 让当前线程放弃当前cpu资源(放弃的时间不确定)
  • setPriority(n): 设置线程优先级(1~10)
  • setDaemon: 设置线程是否为守护线程

如果被停止的线程处于沉睡(sleep)状态, 则会自动抛出此异常并清除中断标记.

Synchronized

因为线程的执行是异步的, 所以在线程中共享的数据不进行同步处理可能会出现问题.

多线程同步(单互斥区)

经典售票问题: 一个售票机有5个窗口同时进行售票, 总量10000张.

它的结果不仅顺序不对, 而且票数计算也可能不对.

synchronized关键字可以解决这个问题, 它的本质是对象锁, 可以修饰普通方法, 静态方法和代码块.

它的作用是将同一个对象中的代码块设定为互斥区, 底层实现原理是信号量.

当进入synchronized代码块时, 其他线程执行到对象锁的代码块时会等待.

synchronized支持锁重入和锁异常, 通俗来说在多线程中使用了synchronized即同步, 不使用即异步.

锁重入: 当一个线程得到一个对象锁时, 再次请求此对象时, 可以再次得到该对象的锁.(支持继承体系)

锁异常: 当一个线程执行的代码出现异常时, 其所持有的锁会自动释放.

###多线程同步(多互斥区)

利用synchronized(obj)在类中制造多个不同的互斥区(2个售票机各售5000张票)

由于Java常量池缓存功能, 锁对象一般直接使用Object而不使用String, Integer等对象.

多线程同步(方法锁和类锁)

synchronized修饰普通方法时, 类中所有的普通方法的锁(方法锁)是synchronized(this).

synchronized修饰静态方法时, 类中所有的静态方法的锁(类锁)是synchronized(TicketThread.class).

方法锁和类锁不是同一把对象锁

关于volatile

使用volatile关键字可以增加实例变量在多个线程之间的可见性, 但它不支持原子性.

JVM在-server模式为了线程的运行效率, 一直在私有堆栈中取值.(这时需要使用volatile增加可见性)

synchronized也实现了volatile的功能, 所以它不仅具有互斥性, 也具有可见性.

线程间通信

等待/通知机制

等待/通知机制通过调用对象锁的wait和notify方法实现.(必须在对应锁对象的同步代码块中使用)

锁对象调用wait方法后线程进入等待状态并释放对象锁(等待状态调用interrupt方法会出现异常)

锁对象调用wait(n)方法后线程进入等待状态, 但如果超过指定时间则会自动唤醒.

锁对象调用notify方法会通知同一锁对象的随机一个等待状态的线程进入就绪状态但不释放对象锁

锁对象调用notifyAll方法会通知同一锁对象的所有等待状态的线程进入就绪状态但不释放对象锁

管道流

管道流是一种特殊的流, 用于不同线程中传送数据.

一个线程发送数据到输出管道, 而另一个线程从输入管道中读取数据.

字节流

如果传送的数据是字节流, 则需要使用PipedInputStream和PipedOutputStream这两个类.

连接输入和输出管道后进行传输, 当输出管道没有数据写入时处于输入管道处于阻塞状态.

字符流

如果传送的数据是字节流, 则需要使用PipedReader和PipedWriter这两个类.

用法与管道的字节流一致, 只是传输的数据类型不同.

join

join方法的主要作用是等待指定的线程对象销毁后再执行接下来的代码

join(n)类似于sleep(n), 区别是前者释放锁, 而后者不释放(内部使用wait(n)实现).

其他

重入锁

ReentrantLock

ReentrantLock是比synchronized更灵活的互斥锁类, 进入锁使用的是lock方法, 而释放锁用的是unlock方法.

ReentrantLock可以配合多个Condition(等待/通知机制的加强类)来解决更复杂的问题.

而Condition的await和singnal方法分别对应于等待/通知机制的wait和notify方法

ReentrantReadWriteLock

ReentrantReadWriteLock是一种用来提高效率读写锁, 它在读时共享, 在写时互斥.

读锁线程共享, 写锁线程互斥.

公平锁

公平锁是指线程获取锁的顺序是按照加锁的FIFO顺序来的, 可以调用isFair方法来判断是否是公平锁.

在Java中默认是非公平锁, 但可以利用ReentrantLock(true)构造方法可以实现公平锁.

线程组

线程组(树型数据结构)中可以有线程对象, 也可以有线程组对象.

可以把线程归属到某一个线程组中, 从而批量管理线程或线程组对象.
