1.4动态SQL

if 元素

尝尝与 test 属性联合使用。

场景：根据角色名称去查找角色，但是角色名称是一个选填条件，不填写是，就不要用它作为条件查询。

choose、when、otherwise 元素

类似 switch…case…default… 功能的语句。

场景：

  • 如果角色编号不为空，则只用角色编号作为条件查询
  • 当角色编号为空，而角色名称不为空，则用角色名称作为条件进行模糊查询
  • 当角色编号和角色名称都为空，则要求角色备注不为空

trim、where、set 元素

去掉 1=1 ,当 where 元素里面的条件成立时，才会加入 where 进 SQL，否则就不加入。

去掉一些特殊 SQL 语法增加前缀后缀，比如开头的 and or 等。
prefix 代表语句的前缀
prefixOverrides 代表去掉第一个什么
suffix 代表后缀
suffixOverrides 代表去掉最后一个什么

set 元素，遇到 , 会去掉

可用 trim 代替

foreach 元素

循环语句，遍历集合，支持数组和 List、Set 接口的集合。往往用于 in 关键字。

  • item 循环中当前的元素
  • index 当前元素在集合的位置下标
  • collection 传进来的参数名称
  • open 和 close 以什么符号讲这些集合元素包装起来
  • separator 元素间隔符

对长度有限制，要预估一下长度。

用 test 的属性判断字符串

作用相当于判断真假，判断空和非空。

bind 元素

提高代码的可移植性，如 MySQL 与 Oracle 模糊查询时，MySQL 可以同时连接多个，而 Oracle 要嵌套除非用||连接多个字符串。

  • name 为生成结果的参数名
  • value 里的参数是传进来的参数
