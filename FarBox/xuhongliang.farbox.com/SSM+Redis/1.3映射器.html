自动映射和驼峰映射

自动映射

  • setting 默认为 true
  • resultType 为 POJO（或强大的 resultMap）
  • SQL 中别名为 POJO 上的属性名称保持一致，就会自动映射

驼峰映射

  • setting 默认为 false
  • SQL 中会按照驼峰命名的方式做自动映射，例，role_name -> roleName
  • 不灵活优缺点，会采用 resultMap 这个强大的映射规则优化

使用 resultMap 映射结果集

自动映射和驼峰映射不够强大无法定义多的属性。

  • resultMap id为它的标识，type代表使用那个类作为映射类，可以是别名或全限定名。
  • 子元素 id代表他的主键，result代表他的其他属性。
  • 子元素属性 property代表POJO的属性名称，column代表SQL的列名，由此建立映射关系。
  • select 中 resultMap 指定了那个 resultMap 作为其映射规则。

概述

可以自定义 typeHandler ，避免数据库异常，如数据库字段返回为 null 。

我们还可以直接指定对应的 jdbcType，从而让 MyBatis 能够探测使用那个 typeHandler 进行转换，避免空指针异常。

如果返回的转换为浮点类型，还可以控制精度。

jdbctype和javatype用处

使用MyBatis框架做更新操作时，在该字段需要更新的内容为空时，就会出现1111错误，也就是无效的列类型，这个时候你就要使用jdbcType。至于什么时候要使用到javaType我还没遇到过，而且我也没有听说过要使用javaType。

存储过程参数支持

输入参数是外界需要传递给存储过程的；
输出参数是存储过程处理后返回的；
输入输出参数一方面外界需要可以传递给它，另一方面最后存储过程也会将它返回给调用者。

级联

MyBatis 中的级联

  • 鉴别器：一个根据某些条件决定采用具体实现类级联的方案，比如体检表要根据性别去区分。
  • 一对一：学生证和学生就是一种一对一的级联。
  • 一对多：班主任和学生就是一种一对多的级联。
  • 没有多对多的级联，因为比较复杂，使用困难，可以通过两个一对多级联进行替换。

雇员表
女性体检表、男性体检表
工牌表
员工任务表
任务表

级联模型：

  • 雇员表为中心
  • 雇员表和工牌表，一对一
  • 雇员表和员工任务表，一对多
  • 员工任务表和任务表，一对一
  • 体检表看雇员性格，会有不同的关联表，一对一
