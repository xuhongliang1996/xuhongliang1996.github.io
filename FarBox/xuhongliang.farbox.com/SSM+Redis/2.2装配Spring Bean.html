2.2装配Spring Bean

依赖注入的3种方式

构造器注入

大部分情况下，我们都是通过类的构造方法来创建类对象，Spring 也可以采用反射的方式，通过使用构造方法来完成注入。

构造器配置

  • id 为别名，class 为注入的类
  • constructor-arg 用于定义类构造方法的参数，index 用于定义参数的位置，value 设置值。

缺点，参数很对，会比较复杂。

使用 setter 注入 *

Spring 通过反射调用没有参数的构造方法生成对象，同时通过反射对应的 setter 注入配置的值。

接口注入

有些时候资源并非来自于自身系统，而是来自于外界，可以采用接口注入的形式来获取它。

装配 Bean 概述

Spring 中提供了3种方法进行装配

  • 1.在 XML 中显示配置
  • 2.在 Java 的接口和类中实现配置
  • 3.隐式 Bean 的发现机制和自动装配原则

开发中优先使用 3 其次 2 最后 1

通过 XML 配置装配 Bean

使用 XML 装配 Bean 需要引入对应的 XML 模式（XSD），这些文件会定义配置 Spring Bean 的一些元素。

beans 为根元素。

装配简易值

  • id 属性是 Bean 的编号，如果没有 Spring 会采用全限定名#{number}的格式生成编号。
  • class 一个类限定名
  • property 定义类的属性
    • name 属性名称
    • value 值

有时候需要注入一些自定义的类，要先定义，在引用。

注入一个 bean 到类的对象属性里。

  • name 属性名称（对象）
  • ref 通过 id 引用对应的 bean

通过注解装配 Bean *

注解既能实现 XML 的功能，也提供了自动装配的功能，程序员所需要做的决断就少了。
提供了两种方式来让容器发现 Bean：

  • 组建扫描：通过定义资源的方式，让 Spring IoC 容器扫描对应的包，从而装配起来。
  • 自动装配：通过注解定义，使得一些依赖关系可以通过注解完成。

公共资源使用 XML 配置，其他的使用注解。

使用 @Component 装配 Bean

在 POJO 中使用注解完成装配。

使用 @Component 后这个类会被扫描生成 Bean 实例，value 类似 bean 中的 id，如果省略，那么就会用类名代替（第一个字母小写）。

此时，还不能进行测试，sping ioc 不知道去哪里扫描对象，可以使用一个 java config 来告诉它

注意：包名和 POJO 要一致，@ComponentScan 代表扫描，此时他会扫描当前包的路径。

有了 POJO 有了 java config 就可以生成 IoC 容器了：

弊端：

  1. 它只能扫描所在包的 Java 类
  2. 只是注入了一些简单值，没有注入对象

解决第一个问题：
增加 @ComponentScan 的配置项

  1. basePackages：它可以配置一个 Java 包的数据，Spring 会根据它的配置扫描对应的包和子包，将配置好的 Bean 装配起来
  2. basePackageClasses：它可以配置多个类，Spring 会根据配置的类所在的包，为包和子包进行扫描装配对应配置的 Bean。

自动装配——@Autowired *

Spring 生成所有的 Bean 后，如果发现这个注解，他就会在 Bean 中查找，然后找到对应的类型，将其注入进来，就完成依赖注入了。

这个 role 需要按类型注入，IoC 容器就会寻找资源，然后将 Role 注入。

@Autowired(required = false)，这个属性会让 Spring 找不到对应类型时不发出异常，字段可能为空。

@Autowired 不仅能进行属性配置，还可以用来进行方法配置。

自动装配的歧义性（@Primary和@Qualifier）

因为要按类型的方式对应一个实现类。

如果一个 RoleService 接口有两个实现类，此时他在某个类中为接口类型，当要自动装配时就会出现问题，他不知道使用哪个实现类进行装配，于是就会抛出异常，这样 @Autowired 注入就失败了。

1. 注解 @Primary
在类的上面定义，定义为优先使用该类注入，如果这是两个实现类中的一个，加上这个后，他就会注入这个实现类。

缺点：只解决首要性问题，不能解决选择性问题。
2. 注解 @Qualifier
在自动装配的下面使用，可以指定要注入的实现类。

装载带有参数的构造方法类

可在带有参数的构造函数中使用@Autowired 或 @Qualifier

使用 @Bean 装配 Bean

@Component 不能注解在方法上，此时可以使用 @Bean ，它可以注解到方法之上，并且将方法返回的对象作为 Spring 的 Bean ，存放在 IoC 容器中。

也可以通过注解注入别的 Bean 中。

注解自定义 Bean 的初始化和销毁方法

  • name 字符串数组，允许配置多个 BeanName
  • autowire 标志是否是一个引用的 Bean 对象
  • initMethod 自定义初始化方法
  • destroyMethod 自定义销毁方法

装配的混合使用

第三方的包或者其他外部的接口，建议使用 XML 的方式进行装载。

@ImportResource
配置的内容是一个数组，可以配置多个 XML 配置文件，就可以引入多个 XML 所定义的 Bean 了。

@Import
用来注入配置类，可加载多个配置文件。

XML 中，一个 XML 文件去引入其他的 XML 文件，XML 不支持加载配置类。

自动扫描，通过 XML 的配置扫描注解包

条件化装配 Bean

在某些条件下不需要去装配 Bean ，我们就要通过条件化去判断，用 @Condition 去配置。

它调用一个类实现接口 Condition ，实现方法先获取上下文环境，判断是否存在关于数据源的基础配置。
