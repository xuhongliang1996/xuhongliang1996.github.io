3.java继承

类、超类和子类

关键字 extends 表明正在构造的新派生类生于一个已存在的类。
已存在的类成为超类、基类或父类；
新类成为子类、派生类或孩子类。

在面向对象程序设计中，应该将通用的方法放在超类中，将具有特殊用途的方法放在子类中。
若超类有些方法不适应子类，子类可以提供一个新的方法来覆盖超类中的这个方法。
但是，如果该方法用到了超类的私有域，此时可以在子类的新方法中调用超类的方法来（通过关键字 super 来调用超类方法），在进行后面的更改实现自己的需求。

super 的另一个使用，在构造器中的应用。
如果要在子类中调用超类的构造函数，就要在子类的构造函数的第一行写上super();，此时会调用默认构造函数，调用带参数的需要加上参数。

继承层次

可以连续继承，但不能一个子类继承多个超类，从某个特定类到其祖先的路径被称为该类的继承链。

多态

在java语言中，对象变量是多态的。超类变量可以引用一个超类对象，也可以引用一个超类的任何子类的对象。
超类变量引用超类的子类，只能使用子类重写超类的方法，不能使用子类特有的方法。

不能将一个超类的引用赋给与类变量，但是子类的引用可以转换成超类引用，而不需要采用强制类型转换。

动态绑定

调用对象方法的执行过程

  1. 编译器查看对象的声明类型和方法名。可能存在多个方面名相同的方法。此时，编译器获取了所有可能被调用的候选方法
  2. 查看调用方法时提供的参数类型。此时，以获取需要调用的方法名字和参数类型
  3. 如果是private方法、static方法、final方法或者构造器，则编译器可以准确的知道调用那个方法。调用方式成为静态绑定
  4. 虚拟机会调用对象实际类型最合适的那个类的方法，如果实际类型没有这个类，则去超类中寻找

阻止被继承：final 类和方法

当类使用 final 时，就表示这个类不能扩展：

final 类被声明后，这个类的方法全会声明为 final ，它的子类就不能覆盖这个类的方法。

使用final在方法上的主要目的是：确保他们不会在子类中改变语义。

强制类型转换

就像需要将浮点型数值强制转换为整型数值一样，也可以将某个类的对象引用转换成另外一个类的对象引用。

原因：在暂时忽视对象的实际类型之后，使用对象的全部功能。
将一个值存入变量时，编译器将检查是否允许该操作。

  • 将一个子类的引用赋给一个超类变量，编译器是允许的。
  • 将一个超类的应用赋给一个子类变量，必须进行类型转换，这样才能通过检察。

应用场景：一般情况下不需要进行类型转换，实现多态性的动态绑定机制能够自动地找到相应的方法，只有在超类变量调用子类引用并要使用子类的特有方法时，才需要进行类型转换。

如果重新设计一个超类，添加子类特有的方法，实现重写，完全可以避免使用强制类型转换。

抽象类

当祖先类更加通用，人们只将它作为派生其他类的基类，而不想作为使用的特定的实例类，这是就可以使用关键字 abstract 关键字，将超类定义为抽象类。
注意：虽然抽象类不能实例化，但作为超类，它可以使用超类变量引用子类对象。

当超类的两个子类有一个同名的方法，子类实现这个方法很简单，但超类不好给方法提供内容，可以让超类的这个方法返回一个空字符串。然而，还有一个更好的方法，就是使用 abstract 关键字，讲这个方法定义为抽象方法。
只要一个类包含抽象方法，那么这个类就对自动成为抽象类，不能被实例化。

继承抽象类的子类，如果不是想抽象类的抽象方法，也会编程抽象类。

受保护访问

子类不能访问超类的私有域，如果想要允许，可以使用 protected 修饰符。
不过子类调用的这个受保护方法只能访问超类被调用受保护方法的方法域，不能访问超类的其他域。这种机制有助于避免滥用受保护机制，是的子类只能获得访问受保护域的权利。

控制可见性的修饰符：

  • 仅对本类可见——private
  • 对所有类可见——public
  • 对本包和所有子类可见——protected
  • 对本包可见——默认

Object：所有类的超类

Object 类是所有类的始祖，每个类都由它扩展而来，他们是默认继承的，不需要写出来。
没有明确的指出超类，那么这个类的超类就是 Object 类。

java 中只有基础类型不是对象，如，数值、字符和布尔类型的值都不是对象。
不管对象数组还是基本类型数组都扩展于 Object 类。

equals 方法 hashCode 方法

equals()相等的两个对象，hashcode()一定相等，equals()不相等的两个对象，却并不能证明他们的hashcode()不相等。
hashcode()不等，一定能推出equals()也不等；hashcode()相等，equals()可能相等，也可能不等。

当从HashSet集合中查找某个对象时，Java系统首先调用对象的hashCode()方法获得该对象的哈希码表，查到相同的哈希表，然后根据哈希吗找到相应的存储区域，最后取得该存储区域内的每个元素与该对象进行equals方法比较。

这个类的两个实例对象用equals()方法比较的结果相等时，他们的哈希码也必须相等；也就是说，如果obj1.equals(obj2)的结果为true,那么以下表达式的结果也要为true:obj1.hashCode() == obj2.hashCode()

toString 方法

用于返回对象值的字符串。这个方法在很多地方都会自动调用。

  • 使用+连接字符串
  • 打印输出到控制台

为每个类重写 toString 方法，程序员可以通过日志记录支持中受益匪浅。

泛型数组列表

java 能在运行时确定数组的大小。

但是并没有完全解决运行时动态更改数组的问题。一旦确定了数组的大小，改变就不太容易了。

访问数组列表元素

虽然数组列表自动扩展容量，但是增加了访问元素语法的复杂程度。
使用 set 和 get 方法实现改变和访问数组元素的操作。

集合添加完毕后，可以使用toArray方法，将数组列表元素拷贝到一个数组中。

使用带索引的 add 方法，可以在数组列表的中间添加元素，后面的元素都要向后退一格。

也可以删除一个元素，删除之后所有后面的元素向前移动一格。

对象包装器与自动装箱

有时需要将基本类型转换为对象，这是就要用到包装器。
有：Integer、Long、Float、Double、Short、Byte（这6个派生于超类 Number）、Character、Void和Boolean
特点：对象包装器类是不可变的，一旦构造了包装器，就不允许更改包装在其中的值。同时，对象包装类还是fianl，因此不能定义他们的子类。

自动装箱：当数组裂变添加基础类型对象时，会自动打包成包装器对象。

自动拆箱：当一个Integer对象赋值给一个int值时，会自动拆箱变成基本类型。

自动拆箱自动装箱：执行算术表达式时，会先拆箱，再执行运算，最后在装箱。

继承设计的技巧

  1. 将公共操作或域放在超类。
  2. 不要使用受保护的域。
  3. 使用继承实现“is-a”关系。
  4. 除非所有继承的方法都有意义，否则不要使用继承。
  5. 在覆盖方法时，不要改变预期的行为。
  6. 使用多态，而非类型信息。
  7. 不要过多的使用反射。
