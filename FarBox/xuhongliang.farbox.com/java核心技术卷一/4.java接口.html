4.java接口

接口

接口不是类，而是对类的一组需求描述，这些类要遵从接口描述的统一格式进行定义。

接口中的所有方法自动的属于public
接口不能提供的功能：

  • 不能含有实例域
  • 不能在接口中实现方法

可以将接口看成是没有实例域的抽象类，但还是有一定区别。

实现了接口的方法。

接口的特性

接口不是类，不能使用new实例化，但是可以声明接口，可以使用instance检查一个对象是否实现了某个特定的接口。

要让类实现某个接口，使用关键子 implements ，可以继承多个接口

接口也可以使用extends扩展接口，并且不像类，它可以扩展多个。

接口不能包含实例域或静态方法，却可以包含常量，被自动设为public static final

接口与抽象类

一个类只能扩展一个类，不能扩展第二个类，而接口可以扩展多个。

对象克隆

拷贝一个变量时，引用同一个对象，更改一个另一个也受影响。
如果不想这样，可以使用 clone 方法。

clone 方法没有这么简单，clone 是 Object 类的方法。
如果对象都是基本类型，拷贝域没有任何问题。
如果对象中包含了对象的引用，他们会引用同一个子对象，与原对象共享这部分信息。
想要改变，就要重写。

接口与回调

回调，可以指出某个特定事件发生时因该采取的动作。

  1. Class A调用Class B中的X方法
  2. Class B中X方法执行的过程中调用Class A中的Y方法完成回调

内部类

内部类是定义在另一个类中的类，原因：

  1. 内部类方法可以访问该类定义所在的作用域中的数据，包括私有数据。
  2. 内部类可以对同一个包中的其他类隐藏起来。
  3. 想要定义一个回调函数且不想编写大量代码时。

使用内部类访问对象状态

内部类方法可以访问该类定义所在的作用域中的数据，包括私有数据。

内部类的特殊语法规则

内部类访问外部类，的复杂形式Outer.tis.属性
外部类访问内部类，的形式Inner inner = new Inner()
其他类访问外部类中的内部类，的形式Outer.Inner inner = new Outer().new Inner()

局部内部类

局部内部类可以对外界完美的隐藏起来。
我们将内部类移到了外部类的方法中，然后在外部类的方法中再生成一个内部类对象去调用内部类方法，这就是局部内部类。

由外部方法访问 final 变量

局部内部类，不仅能够访问包含他们的外部类，还可以访问局部变量，但必须被声明为 final。

匿名内部类

加入只创建这个类的一个对象，就不必命名了，这种了被称为匿名内部类。
类名可以是一个接口，于是内部类就是实现这个接口；
也可以是一个类，于是内部类就是对这个类扩展。

静态内部类

有时，使用内部类知识为了把一个类隐藏在另外一个类的内部，并不需要内部类引用外围类对象。
可以将内部类声明为 static，以便取消产生的引用。
如果用static 将内部内静态化，那么内部类就只能访问外部类的静态成员变量，具有局限性。
其次，因为内部类被静态化，因此Out.In可以当做一个整体看，可以直接new 出内部类的对象（通过类名访问static，生不生成外部类对象都没关系）。
